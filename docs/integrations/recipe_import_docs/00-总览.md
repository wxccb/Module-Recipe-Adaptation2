# Codex MM Recipe Bridge 配方导入总览

本整合包内的统一桥接模组为 `codex-mm-recipe-bridge-1.3.0.jar`，modid 是 `codexmmrecipebridge`。

它不会创建任何 Modular Machinery 机器，只注册“配方适配器”。你需要在 ZenScript 中用 `RecipeAdapterBuilder` 选择某台 MM/MMCE 机器要导入哪个原模组机器的最终运行时配方。

## 工作方式

- 读取的是游戏加载完成后的运行时配方表：CraftTweaker/ModTweaker/脚本增加、删除、替换后的结果会被导入。
- 一个 adapter id 对应一类原模组机器，例如 `ic2:macerator` 或 `thermalexpansion:pulverizer`。
- 原配方有概率副产物时，导入为 MM/MMCE 的 chanced output；可以用 `RecipeModifierBuilder` 的 `affectChance = true` 放大概率。
- 输入/输出/流体/能量/时间都会作为 MM recipe requirement 写入，可用 MMCE/扩展写法继续追加 requirement、tooltip、事件等。
- AE2 只导入 AE2 公共 API 暴露的机器配方：研磨机与压印器；充能器只有充电倍率注册表，不是普通机器配方表。

## 基础脚本

```zenscript
import mods.modularmachinery.RecipeAdapterBuilder;
import mods.modularmachinery.RecipeModifierBuilder;

// 把 IC2 打粉机最终配方导入 your_machine
RecipeAdapterBuilder.create("your_machine", "ic2:macerator").build();

// 输入翻倍、输出翻倍
RecipeAdapterBuilder.create("your_machine_x2", "thermalexpansion:pulverizer")
    .addModifier(RecipeModifierBuilder.create("modularmachinery:item", "input", 2.0, 1, false).build())
    .addModifier(RecipeModifierBuilder.create("modularmachinery:item", "output", 2.0, 1, false).build())
    .build();

// 只放大概率副产物/概率流体输出：最后一个参数 true 表示 affectChance
RecipeAdapterBuilder.create("your_machine_chance", "enderio:sag_mill")
    .addModifier(RecipeModifierBuilder.create("modularmachinery:item", "output", 2.0, 1, true).build())
    .build();

// 流体输入/输出翻倍
RecipeAdapterBuilder.create("your_fluid_machine", "thermalexpansion:fractionating_still")
    .addModifier(RecipeModifierBuilder.create("modularmachinery:fluid", "input", 2.0, 1, false).build())
    .addModifier(RecipeModifierBuilder.create("modularmachinery:fluid", "output", 2.0, 1, false).build())
    .build();
```

## 注意

- adapter id 必须在脚本加载阶段注册；目标 MM 机器必须已经存在。
- 如果某个 addon 的机器没有稳定公开配方表，本桥接不会猜内部实现，以免启动崩溃或导入错误。
- 对已有导入配方做倍化，推荐在 `RecipeAdapterBuilder` 上加 modifier，而不是复制原模组配方。
